Этот скрипт автоматически собирает данные из нескольких архивов и добавляетс данные из них в одну общую базу, сохраняя всю структуру оборудования и файлы спектров.

1. Подготовка архивов
Соберите все .zip архивы с выгруженными базами и положите их в папку bak_files, которая находится прямо рядом со скриптом.

2. Запуск и настройка
Запустите файл main.py. В консоли появятся три вопроса с настройками. Вы можете ввести свои значения или просто три раза нажать Enter, чтобы согласиться со стандартными настройками:

Имя SQL сервера: (по умолчанию .\VDE_SQL_SERVER).

Имя целевой базы: как будет называться ваша итоговая объединенная база (по умолчанию Target_base).

Рабочая папка: место на диске, где сервер физически разместит файлы базы данных и куда выгрузятся файлы спектров (по умолчанию C:\SQLwork).

Шаг 3. Процесс слияния
Дальше скрипт всё сделает сам:

Если целевой базы еще нет, он возьмет первую бд, проверит её и сделает из нёё основную.

Затем он по очереди распакует все оставшиеся архивы и добавит из них предприятия, цеха, агрегаты и измерения. Никаких дубликатов не должно появится.

Все файлы осциллограмм (.afdata), которые были спрятаны глубоко внутри архивов, скрипт вытащит и сложит в папку waves внутри вашей рабочей директории.

Шаг 4. Завершение
Дождитесь сообщения: «ИНТЕГРАЦИЯ ВСЕХ БАЗ ЗАВЕРШЕНА».

Аудит данных и аналитика
Помимо основного алгоритма интеграции, в проекте предусмотрены инструменты для проверки и анализа полученного результата:

Аудит целостности (db_validator.py)
Скрипт для проверки качества и структурной целостности итоговой базы. Запускается отдельно. Анализирует БД на предмет наличия записей, потерянных связей в иерархии оборудования, а также выявляет исторические аномалии, перенесенные из исходных баз (пустые замеры, дубликаты записей по времени).

Оценка рисков (risks_analitic.txt)
Документ содержит описание потенциальных проблем и коллизий, которые могут возникнуть при слиянии данных из распределенных источников. Описаны риски совпадения имен объектов, переноса аппаратных ошибок и влияние человеческого фактора.

Аналитические запросы (requests.txt)
Текстовый файл с подготовленным набором SQL-запросов для сбора статистической информации (количество оборудования, объем данных спектров, статистика инцидентов). Запросы предназначены для самостоятельного выполнения в среде SQL Server Management Studio (SSMS) или DBeaver при подключении к целевой базе.
